HTML Basics

What HTML is, how a page is structured, and how tags work.

What is HTML?

HTML (HyperText Markup Language) is the standard markup language for creating web pages. It describes the structure and content of web documents using elements. Browsers read HTML and render it as visual pages.

Key idea: HTML is about meaning (structure), not style. CSS handles looks, JS handles behavior.
Anatomy of an Element

An element has an opening tag, content, and a closing tag. Some are self-closing. 

<!-- Normal element -->
<p>Hello, world!</p>

<!-- Self-closing (void element) -->
<br />
<img src="photo.jpg" alt="A photo" />
The Boilerplate

The basic structure of an HTML document. Every HTML page starts with this skeleton: 

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8"/>
    <meta name="viewport" content="width=device-width, initial-scale=1.0"/>
    <title>Page Title</title>
  </head>
  <body>
    <!-- visible content here -->
  </body>
</html>
Essential tags
  • Block elements — start on a new line, take full width: <div>, <p>, <h1>–<h6>, <section>
  • Inline elements — flow within text: <span>, <a>, <strong>, <em>, <img>
Note: CSS display property can change any element's behavior.
Block vs Inline
  • Block elements — start on a new line, take full width: <div>, <p>, <h1>–<h6>, <section>
  • Inline elements — flow within text: <span>, <a>, <strong>, <em>, <img>
Note: CSS display property can change any element's behavior.

Text Elements

Headings, paragraphs, emphasis, and inline text formatting.

Headings <h1>–<h6>

Six levels of headings. Use one <h1> per page (SEO + accessibility), then h2–h6 hierarchically. 

<h1>Main Title</h1>
<h2>Section</h2>
<h3>Sub-section</h3>
Paragraph <p>

The main block for text. Browsers add top/bottom margin by default. 

<p>This is a paragraph of text.</p>
Emphasis & Strong
Tag Meaning
<strong> Important text (bold visually)
<em> Stressed emphasis (italic)
<b> Bold — no semantic weight
<i> Italic — no semantic weight
<mark> Highlighted / relevant text
<small> Fine print, side comments
<del> Deleted text (strikethrough)
<ins> Inserted text (underline)
<sub> Subscript (H₂O)
<sup> Superscript (x²)
Other Text Tags
Tag Use
<br> Line break (void element)
<hr> Thematic break / divider
<pre> Preformatted text (preserves spaces)
<code> Inline code snippet
<kbd> Keyboard input (Ctrl+C)
<abbr> Abbreviation with title tooltip
<blockquote> Long quotation from another source
<q> Short inline quotation
<cite> Reference to a work title
<address> Contact information
03

Links & Images

Hyperlinking pages and embedding images.

Anchor Tag <a> 
<!-- External link -->
<a href="https://example.com" target="_blank" rel="noopener">
  Visit Example
</a>

<!-- Internal / same-page link -->
<a href="#section2">Jump to Section 2</a>

<!-- Email & phone links -->
<a href="mailto:hi@example.com">Email us</a>
<a href="tel:+1234567890">Call us</a>
Attribute Purpose
href Destination URL (required)
target _blank = new tab, _self = same tab
rel Relationship (noopener for security)
download Triggers file download
Image Tag <img> 
<img
  src="photo.jpg"
  alt="A mountain at sunset"
  width="600"
  height="400"
  loading="lazy"
/>
Attribute Notes
src Path or URL to image (required)
alt Text if image fails — required for a11y
width / height Prevents layout shift (CLS)
loading lazy = defer off-screen images
Always add a meaningful alt attribute for screen readers. Use alt="" for decorative images.
04

Lists

Unordered, ordered, and description lists.

Unordered List <ul>

Bullet points — for items where order doesn't matter.

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
Ordered List <ol>

Numbered — for steps, rankings, sequences.

<ol type="1" start="1">
  <li>Boil water</li>
  <li>Add pasta</li>
  <li>Cook 10 min</li>
</ol>
<!-- type: 1, A, a, I, i -->
Description List <dl>

Term-definition pairs — great for glossaries and FAQs.

<dl>
  <dt>HTML</dt>
  <dd>HyperText Markup Language</dd>
  <dt>CSS</dt>
  <dd>Cascading Style Sheets</dd>
</dl>
Nested Lists

Lists can be nested inside <li> for sub-items. 

<ul>
  <li>Frontend
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Backend</li>
</ul>
05

Tables

Displaying tabular data — never use tables for layout!

Full Table Structure
<table>
  <caption>Student Grades</caption>
  <thead>
    <tr>
      <th scope="col">Name</th>
      <th scope="col">Score</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Alice</td>
      <td>95</td>
    </tr>
    <tr>
      <td>Bob</td>
      <td>88</td>
    </tr>
  </tbody>
  <tfoot>
    <tr>
      <td>Average</td>
      <td>91.5</td>
    </tr>
  </tfoot>
</table>
Spanning Cells

Use colspan and rowspan to merge cells. 

<!-- Span across 2 columns -->
<td colspan="2">Merged cell</td>

<!-- Span across 2 rows -->
<td rowspan="2">Tall cell</td>

Forms

Collecting user input — the backbone of interactive web.

Form Structure <form> 
<form action="/submit" method="post">
  <label for="name">Your Name</label>
  <input type="text" id="name" name="name" required/>
  <button type="submit">Send</button>
</form>
Tip: Always link <label for="id"> to <input id="..."> — it improves accessibility and click area.
All Input Types
type= What it does
text Single-line text
email Email with validation
password Masked text
number Numeric spinner
tel Telephone (no validation)
url URL with validation
date Date picker
time Time picker
range Slider
color Color picker
checkbox Multi-select tick boxes
radio Single choice from group
file File upload
hidden Data sent but not shown
submit Submit button
reset Reset all fields
search Search box
More Form Elements
<!-- Multi-line text -->
<textarea name="message" rows="4"></textarea>

<!-- Dropdown -->
<select name="city">
  <option value="">Choose…</option>
  <option value="nyc">New York</option>
  <option value="lon">London</option>
</select>

<!-- Group related fields -->
<fieldset>
  <legend>Shipping Info</legend>
  <!-- inputs... -->
</fieldset>

Semantic HTML5

Meaningful tags that describe content — critical for SEO and accessibility.

Page Layout Elements
<header>  ← site/section header, nav, logo
<nav>     ← navigation links
<main>    ← primary content (one per page)
<article> ← self-contained content (blog post)
<section> ← thematic grouping with a heading
<aside>   ← sidebar / tangentially related
<footer>  ← copyright, links, contact
Rule of thumb: Could you replace this with a <div> and lose meaning? Then use the semantic tag. Could not? Use <div>.
Semantic Inline & Content
Tag Meaning
<time> Date/time with machine-readable datetime attr
<figure> Self-contained media (image, chart, code)
<figcaption> Caption for a <figure>
<details> Expandable disclosure widget
<summary> Visible heading for <details>
<dialog> Modal or popup dialog box
<progress> Task progress bar
<meter> Scalar measurement within range
Typical Page Structure
<body>
  <header>
    <nav>…</nav>
  </header>

  <main>
    <article>
      <h1>Post Title</h1>
      <section>…</section>
    </article>
    <aside>Related links</aside>
  </main>

  <footer>© 2026</footer>
</body>

Media Elements

Audio, video, iframes, and responsive images.

Video <video> 
<video
  controls
  autoplay
  muted
  loop
  width="640"
  poster="thumb.jpg"
>
  <source src="video.mp4" type="video/mp4"/>
  <source src="video.webm" type="video/webm"/>
  Your browser doesn't support video.
</video>
Audio <audio> 
<audio controls>
  <source src="track.mp3" type="audio/mpeg"/>
  <source src="track.ogg" type="audio/ogg"/>
</audio>
Iframe <iframe>

Embeds another page (YouTube, maps, etc.).

<iframe
  src="https://www.youtube.com/embed/xyz"
  width="560"
  height="315"
  allowfullscreen
  loading="lazy"
  title="Tutorial video"
></iframe>
Responsive Images <picture>

Serve different images based on screen size or format support. 

<picture>
  <source
    srcset="hero-large.webp"
    media="(min-width: 800px)"
    type="image/webp"
  />
  <source srcset="hero-small.jpg"/>
  <img src="hero-small.jpg" alt="Hero"/>
</picture>

Head & Meta Tags

Everything that goes in <head> — invisible but vital.

Essential Head Tags
<head>
  <!-- Character encoding -->
  <meta charset="UTF-8"/>

  <!-- Responsive viewport -->
  <meta name="viewport"
        content="width=device-width, initial-scale=1.0"/>

  <!-- Page title (tab + SEO) -->
  <title>My Awesome Page</title>

  <!-- SEO description -->
  <meta name="description"
        content="A short description for search engines."/>

  <!-- Open Graph (social sharing) -->
  <meta property="og:title" content="Page Title"/>
  <meta property="og:image" content="preview.jpg"/>

  <!-- Link CSS -->
  <link rel="stylesheet" href="styles.css"/>

  <!-- Favicon -->
  <link rel="icon" href="favicon.ico"/>

  <!-- JS (defer = load after HTML) -->
  <script src="app.js" defer></script>
</head>

Global Attributes

These attributes work on any HTML element.

Attribute Purpose
id Unique identifier — used by CSS, JS, and anchor links
class One or more CSS class names (space-separated)
style Inline CSS — use sparingly
title Tooltip on hover
lang Language of element's content
dir Text direction: ltr / rtl / auto
hidden Hides element (like display:none)
tabindex Keyboard focus order (0 = natural, -1 = skip)
contenteditable Makes element directly editable
draggable Enable HTML drag-and-drop API
spellcheck Enable/disable browser spell check
data-* Custom data attributes for JS hooks
aria-* Accessibility roles and properties
role ARIA landmark role (button, alert, …) 
<!-- data-* example -->
<button
  data-product-id="42"
  data-category="shoes"
>Add to cart</button>

<!-- Access in JS -->
btn.dataset.productId  // "42"

Best Practices

Write HTML that's accessible, performant, and maintainable.

✅ Accessibility (a11y)
  • Always include alt text on images
  • Use <label> linked to every input
  • Use semantic tags, not just <div>
  • Add role and aria-* where needed
  • Ensure keyboard navigation works (tabindex)
  • Use sufficient color contrast in CSS
  • Avoid autoplay without muted
⚡ Performance
  • Add width/height on images (prevents CLS)
  • Use loading="lazy" for off-screen images
  • Put <script> at end of body or use defer
  • Use <link rel="preload"> for critical assets
  • Prefer WebP over JPEG/PNG for images
  • Minimize inline styles
🧹 Clean Code
  • Use lowercase for all tags and attributes
  • Always quote attribute values
  • Close all tags (even optional ones)
  • Indent nested elements consistently (2 spaces)
  • Use meaningful id/class names (BEM methodology)
  • Validate at validator.w3.org
🔍 SEO Essentials
  • One <h1> per page — include main keyword
  • Descriptive <title> (50–60 chars)
  • Meaningful meta description (150–160 chars)
  • Use semantic HTML (search engines read structure)
  • Add lang attribute to <html>
  • Use descriptive alt on content images
